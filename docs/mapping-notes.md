# 구현 메모: 법제처 API 차이 흡수

## 왜 Provider 인터페이스가 필요한가
법제처 API는 서비스별로 인증 파라미터명(`OC`, `key` 등), 검색 파라미터(`query`, `q`), 응답 포맷(JSON/XML), 루트 필드명이 다릅니다.

MCP 외부 인터페이스(`search_law`, `get_law_article`)는 고정하고,
내부에서 `LawProvider` 구현체만 교체하면 API 차이를 흡수할 수 있도록 설계했습니다.

## 실제 연결 시 해야 할 일
1. 검색 API는 공식 가이드(`target=law`, `mobileYn=Y`, `query`, `display`)를 반영했다.
2. 상세/조문 조회(`lawService.do`)는 계정별 권한/응답 차이가 있어, 실계정 응답으로 필드 키를 확정해야 한다.
3. 인증 실패 시 보통 "사용자 정보 검증에 실패" 메시지가 반환되며, 등록된 IP/도메인 화이트리스트 확인이 필요하다.
