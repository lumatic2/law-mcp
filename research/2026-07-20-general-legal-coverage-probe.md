# law-mcp 범용 법률 도메인 커버리지 실측

- 작성: 2026-07-20
- 소비처: horizon `general-legal-coverage` (신규 개설 예정)
- 방법: 실 API(`LAW_API_OC` 실키) 직접 호출. 출처 = 법제처 국가법령정보 공동활용 DRF
  (`https://www.law.go.kr/DRF/lawSearch.do`, 접근일 2026-07-20).
- 목적: Objective 4축(넓이·커버리지·도달·설치 가능성)의 갭을 추정이 아니라 관측으로 확정.

---

## 1. 넓이 — 비세무 도메인에서 무너진다 (가장 큰 갭)

5개 도메인 13개 자연어 쿼리를 `search_law`/`search_precedents` 로 실행.

| 지표 | 값 |
|---|---|
| `search_law` 0건 아님(hit) | 12/13 (92%) |
| **정답 법령이 상위 3에 포함** | **4/13 (31%)** |
| `search_precedents` hit | 9/13 (69%) |

**hit율은 착시다.** 결과가 돌아오지만 내용이 틀렸다:

| 쿼리 | 반환 상위 3 | 정답 |
|---|---|---|
| 부당해고 구제신청 기간 | 병역법 / 선원법 / 의료법 | 근로기준법·노동위원회법 ✗ |
| 정당방위 성립 요건 | 민법 / 관세법 / 전파법 | 형법 ✗ |
| 음주운전 처벌 기준 | 병역법 / 선원법 / 보험업법 | 도로교통법 ✗ |
| 계약 해제 원상회복 의무 | 광업법 / 가사소송법 / 공항시설법 | 민법 ✗ |
| 소멸시효 중단 사유 | 관세법 / 광업법 / 고용보험법 | 민법 ✗ |
| 임대차 보증금 반환 청구 | 공증인법 / 국세기본법 / 국세징수법 | 주택임대차보호법·민법 ✗ |
| 연차휴가 미사용 수당 계산 | 임금채권보장법 시행규칙 / 조세특례제한법 시행규칙 | 근로기준법 ✗ |
| 재산분할 청구권 행사 기간 | 관세법 / 가사소송법 / 공직선거법 | 민법 ✗ |
| 전세권 설정 등기 말소 | **부동산등기법** / 부동산등기규칙 / 가등기담보법 | ✅ |
| 묵시적 갱신 계약 해지 통보 기간 | **주택임대차보호법** / 약관규제법 / 전자금융거래법 시행령 | ✅ |
| 손해배상 예정액 감액 | **민법** / 법인세법 / 국가재정법 | ✅ |
| 친권자 변경 심판 | **민법** / 소년법 / 국제사법 | ✅ |
| 포괄임금제 유효 요건 | (0건) | ✗ (판례 법리라 법령에 없음) |

**원인**: 법령명 매칭이 0건이면 본문검색(`search=2`)으로 폴백하는데, 본문검색은 전체 법령을
대상으로 하고 **관련도 랭킹이 없다**(2026-07-20 ib3에서 확인·기록). 세무 쿼리가 잘 되던 것은
용어가 특이해서(가지급금·매입세액) 매칭 법령 자체가 적었던 덕이고, 일반 법률어(해제·시효·요건)는
수백 개 법령 본문에 흔해 노이즈에 묻힌다. **즉 "세무 특화"가 아니라 "희귀어 특화"였다.**

시사점: 랭킹 개선은 선택이 아니라 범용화의 전제. 그리고 개선을 주장하려면 먼저 **측정 하네스**가
있어야 한다(현재 기준선 = 정답 포함률 31%).

## 2. 커버리지 — 누락 법원(法源) 5종이 전부 실재한다

`lawSearch.do` 에 `query=임대차` 로 타깃별 실측:

| target | 법원 | 응답 컨테이너 | totalCnt | 비고 |
|---|---|---|---|---|
| `prec` | 판례 | `PrecSearch` | 368 | 지원 중 |
| `admrul` | 행정규칙 | `AdmRulSearch` | 15 | 지원 중 |
| `expc` | **법령해석례** | `Expc` | 23 | 미지원 — 안건명·회신기관명·안건번호 |
| `detc` | **헌재결정례** | `DetcSearch` | 42 | 미지원 (필드에 `헌재결정례일련번호`) |
| `decc` | **행정심판재결례** | `Decc` | 7 | 미지원 (필드에 `재결청`·`처분청`) |
| `ordin` | **자치법규(조례·규칙)** | `OrdinSearch` | 48 | 미지원 |
| `lstrm` | **법령용어** | `LsTrmSearch` | 31 | 미지원 |
| `licbyl` | 별표·서식 | `licBylSearch` | 28 | 미지원 |
| `oldAndNew` | 신구법비교 | `OldAndNewLawSearch` | 7 | 미지원 |
| `trty` | 조약 | `TrtySearch` | 0 | 쿼리 특성상 0건 |

⚠ **주의: 타깃명과 법원 대응이 직관과 반대다.** `detc` 가 헌재결정례, `decc` 가 행정심판재결례로
관측됐다(응답 필드 기준). 구현 시 이름이 아니라 **응답 필드로 확정**할 것.

시사점: 추가 비용이 낮다. 검색 응답 구조가 기존 `admrul`/`prec` 과 동형이라 어댑터 패턴 재사용 가능.
`lstrm`(법령용어)은 부수 효과가 큼 — 지금 수기로 관리하는 `term-bridge.ts` 사전의 상위 호환 후보.

## 3. 도달 — 조문 단위 검색 타깃은 존재하지 않는다

`lawjosub`·`lsJo`·`joSub`·`eflawjosub` 전부 **빈 응답**. 즉 "이 쟁점이 어느 법 몇 조인가"를
upstream 한 번으로 묻는 경로가 없다. 현재 `get_law_article` 은 조문 번호를 *이미 알 때만* 쓴다.

가능한 경로: `lawService.do?target=law&ID=…` 로 법령 전문을 받으면 **조문 단위 구조**(조문단위→항→호→목)가
들어 있다(`extractFullArticleContent` 가 이미 파싱 중). 즉 조문 단위 검색은 **클라이언트에서 구성 가능**하다.
비용 실측: 법령 1건 전문 JSON 316KB~580KB, 병렬 5건 1.1초.

시사점: (2)의 랭킹 문제와 (3)의 조문 도달이 같은 해법으로 풀린다 — 후보 법령의 조문을 받아
쿼리와 매칭하면 "어느 법"과 "몇 조"를 동시에 답하고, 그 매칭 점수가 곧 관련도 신호가 된다.
ib3에서 기각된 (B)안(본문 토큰 빈도)의 실패 원인이 "법령 단위 빈도라 의미 불일치를 못 거름"이었으므로,
**조문 단위**로 내려가면 그 반증이 그대로 적용되지는 않는다(조문은 문맥이 좁다) — 단 검증 필요.

## 4. 설치 가능성 — 현재 외부인은 설치할 수 없다

- private 레포(`github.com/lumatic2/law-mcp`), npm 미배포.
- `README.md` 에 MCP 클라이언트 설정 예시(`claude_desktop_config.json`/`.mcp.json`) 없음.
- `LAW_API_OC` 발급 절차 안내 없음 — 법제처 OpenAPI 는 이용자 등록이 필요하다.
- `package.json` 에 `"private": true`, `bin` 엔트리 없음(= `npx` 실행 불가).

시사점: 기술 부채가 아니라 **누락**이다. 다만 품질(1~3) 이후 순서가 맞다 — 설치 가능해진 뒤
품질이 낮으면 첫인상을 잃는다.

## 포화 판정

이 리서치는 **포화로 닫았다** — 마지막 확인(조문 검색 타깃 4종, 배포 상태)이 새 사실을 더하지 않고
기존 결론을 확인만 했다. 미확인으로 남긴 것: MCP 레지스트리·디렉터리 등재 요건(설치 가능성 축의
세부 — LB4 착수 시점에 조사한다).
