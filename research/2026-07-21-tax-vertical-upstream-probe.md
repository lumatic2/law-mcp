## 1. 조세심판원 — 선행 조사의 target 이름이 **뒤집혀 있었다**

`2026-07-21-lawgo-api-survey.md` D-2 는 `specialDeccTt` 로 시도해 빈 응답을 받고
"파라미터·권한 미상"으로 분류했다. **이름 순서가 반대였다.**

```
target=specialDeccTt  → 빈 응답
target=ttSpecialDecc  → totalCnt 4688   ← 실재
```

같은 규칙이 형제 target 에도 적용된다(`{기관}SpecialDecc` 어순):

| target | 프로브 | 건수 |
|---|---|---|
| `ttSpecialDecc` 조세심판원 | "가산세" | **4,688** |
| `adapSpecialDecc` 소청심사 | "처분" | 201 |
| `acrSpecialDecc` 감사원 심사청구 | "처분" | 79 |
| `kmstSpecialDecc` 해양안전심판원 | "처분" | 0 (컨테이너는 응답) |

> **선행 조사의 "존재하지 않음" 판정은 어순 오류였다.** 이 레포가 `lsRlt` 등 다른 빈 응답
> target 을 "없음"으로 정리한 것도 같은 오류일 수 있다 — 다만 이번 horizon 범위 밖이다.

### 세법 쿼리에서의 실제 밀도

| 쿼리 | 결정례 수 |
|---|---|
| 가산세 | 4,688 |
| 1세대 1주택 | 3,885 |
| 접대비 | 225 |
| 이행강제금 | 3 |

### 응답 계약 (어댑터 설계 입력)

**검색** `lawSearch.do?target=ttSpecialDecc&query=…` → `Decc.decc[]`

```json
{"id":"1","재결청":"조세심판원","특별행정심판재결례일련번호":"1704",
 "청구번호":"조심 2018광1070","의결일자":"2018.06.04","재결구분명":"조세",
 "사건명":"10년의 국세부과의 제척기간을 적용하여 부가가치세를 부과한 처분의 당부 및
          부당과소신고가산세가 아닌 일반과소신고가산세 적용대상인지 여부",
 "데이터기준일시":"2024.12.18"}
```

**전문** `lawService.do?target=ttSpecialDecc&ID=1704&type=JSON` → `SpecialDeccService` (8.8KB)

```
재결청 / 청구취지 / 참조결정 / 재결요지 / 이유
```

- `재결요지` = 판례의 판시사항에 해당하는 **요지 필드가 upstream 에 이미 있다.**
  우리가 요약을 만들 필요가 없다 — LB3 의 `nlrc` 판정요지와 같은 구조다.
- ⚠ **ID 는 `id` 필드가 아니라 `특별행정심판재결례일련번호` 다.** `id` 는 행 번호(1,2,3…)이고
  이걸 ID 로 넘기면 엉뚱한 문서가 온다(프로브에서 실제로 발생 — `ID=1` → 무관한 응답).
- ⚠ `type=HTML` 은 **본문이 없다**(1,973B 짜리 JS 로더 셸 — 모바일 ajax 분기). JSON 만 쓴다.
- 데이터 기준일시 2024.12.18 — **1년 7개월 낡았다.** 최신 결정례는 못 준다(한계로 선언할 것).

## 2. 시행일 법령 `eflaw` — 과세연도 축의 재료

```
lawSearch.do?target=eflaw&query=소득세법 → totalCnt 775
{"현행연혁코드":"시행예정","법령명한글":"소득세법","법령ID":"001565",
 "시행일자":"20280101","공포일자":"20241231","제개정구분명":"일부개정",
 "법령상세링크":"…target=eflaw&MST=267581&…&efYd=20280101"}
```

- **`efYd`(시행일자) 파라미터가 상세 링크에 실려 있다** — 시점 지정 조회의 계약이 존재한다.
- `lawService.do?target=eflaw&LM=소득세법` 은 조문 전문을 반환(기본정보·조문·부칙·개정문·제개정이유).
### 과거축도 열린다 — 이 horizon 의 최대 발견

`efYd` 에 **지난 날짜**를 넣어도 그 시점의 조문이 온다. 조문 수가 실제로 다르다:

| 호출 | 법령명 | 시행일 | 공포일 | 조문 수 |
|---|---|---|---|---|
| `target=law&LM=소득세법` (현행) | 소득세법 | 20260101 | 20251223 | **393** |
| `target=eflaw&LM=소득세법&efYd=20230101` | 소득세법 | 20230101 | 20220812 | **338** |
| `target=eflaw&LM=소득세법&efYd=20240101` | 소득세법 | 20240101 | 20221231 | **341** |
| `target=eflaw&LM=소득세법&efYd=20280101` | 소득세법 | 20280101 | 20241231 | 382 |

`eflaw` 검색은 연혁본까지 함께 준다(소득세법 20건 중 시행예정 7·현행 1·연혁 12 —
`20260421(연혁) 20260101(연혁) 20250701(연혁) …`). `현행연혁코드` 로 구분된다.

⚠ **`target=law` 에 `efYd` 를 붙이면 조용히 무시되고 현행이 온다**(위 C 프로브 — 시행일 20260101
그대로). 시점 조회는 반드시 `eflaw` 로 가야 한다. 이 함정을 모르면 "시점 지정했는데 현행 답"이
나오고 **아무 경고도 없다.**

> **이것이 세법 vertical 의 차별점이다.** 도메인 리서치가 확인한 바 조세심판원조차 "귀속년도"를
> 사건 식별 1급 메타데이터로 쓴다(§3). 같은 조문 번호라도 귀속연도가 다르면 요건이 다르므로,
> 시행일자를 안 밝히고 조문만 주는 도구는 **구조적으로 틀린 답을 준다.**
> 확인된 경쟁자(`korean-law-mcp`·삼일·택스넷) 중 귀속연도 인식을 표방한 곳은 없다.

연혁 전용 target 은 죽어 있다: `lsHistory` HTML · `lsHstInf` 0건 · `eflawJo` 빈 응답.
**연혁 접근 경로는 `eflaw` 하나뿐이다.**

## 3. 별표·서식 `licbyl` — 세율표는 파일 뒤에 있다

```
lawSearch.do?target=licbyl&query=세율 → totalCnt 97
{"별표명":"1세대 3주택 이상자의 장기임대주택 등 일반세율 적용신청서",
 "관련법령명":"소득세법 시행규칙","별표종류":"서식","별표번호":"008306",
 "별표서식파일링크":"/LSW/flDownload.do?flSeq=164445073",
 "별표서식PDF파일링크":"/LSW/flDownload.do?flSeq=164445075"}
```

- 메타데이터(별표명·관련법령·번호)는 JSON 으로 온다 → **"어느 별표를 봐야 하는지" 는 지금도 답 가능.**
- 그러나 **표 내용 자체는 파일**(HWP/PDF)이다. 텍스트 추출은 별도 비용이고 실패 가능성이 있다.
- 판정: **메타데이터 출하는 싸고, 내용 추출은 비싸다.** 둘을 한 milestone 에 묶지 않는다.

## 4. 국세청 예규 — **있다. 여기서도 어순이었다**

1차 프로브에서 `cgmExpc` 계열 10종 변형이 전부 실패했다(`cgmExpc1220000`·`ntsExpc`·`expcNts`·
`expc&org=1220000` …). 병렬 도메인 리서치가 문서상 이름 **`ntsCgmExpc`** 를 물어왔고, 실측했다:

```
target=cgmExpc1220000  → 빈 응답
target=ntsCgmExpc      → totalCnt 1938   ← 실재
```

**조세심판원과 똑같은 `{기관}+{서비스}` 어순**이다. 형제도 같은 규칙을 따른다:

| target | 기관 | 프로브("가산세") |
|---|---|---|
| `ntsCgmExpc` | 국세청 | **1,938** |
| `moefCgmExpc` | 기획재정부 | 66 |
| `molabCgmExpc` | 고용노동부 | 빈 응답 (해당 없음) |

### 응답 계약

```json
{"안건명":"’08년 이전 취득 주식에 대해 ‘출연재산가액 1% 이상 직접 공익목적 사용 의무’
          위반시 주식초과 보유분에 대한 가산세 적용 여부",
 "안건번호":"기준-2023-법규법인-0191[법규과-992]", "해석기관명":"국세청",
 "해석일자":"2025.05.15", "법령해석일련번호":"1643869", "해석기관코드":"1210000",
 "법령해석상세링크":"https://taxlaw.nts.go.kr/qt/USEQTA002P.do?ntstDcmId=200000000000012562",
 "데이터기준일시":"2025.06.28"}
```

- **안건번호 체계**(`기준-2023-법규법인-0191`)가 그대로 온다 — 실무자가 인용하는 형식 그 자체다.
- 상세 링크가 법제처가 아니라 **국세법령정보시스템(taxlaw.nts.go.kr)** 을 가리킨다. 이 레포는
  이미 NTS 폴백 경로를 갖고 있다(IB1b) — 본문 확보에 그 선례를 재사용할 수 있다.
- 데이터기준일시 2025.06.28 — 조세심판원(2024.12.18)보다 신선하다.
- ⚠ **`display=1` 일 때 배열이 아니라 단건 객체로 온다**(프로브에서 상세 조회가 조용히 실패했다).
  UD3 에서 위원회 9종에 대해 고친 "단건객체 수리"와 같은 결함 — 어댑터에서 동일 처방 필요.
- 전문(`lawService.do?target=ntsCgmExpc&ID=…`) 계약은 **미확인** — 위 단건객체 결함으로 프로브가
  ID 를 못 뽑았다. TV2 의 첫 검증 항목.

### 다만 예규는 법이 아니다 — 출하 시 등급 표시가 필요하다

도메인 리서치(`2026-07-21-tax-domain-tax-practice-sources.md` §1)가 확인한 바:
예규·기본통칙은 **법원(法院)을 구속하지 않는 "자료"** 이고(대법원 1998.9.8 인용),
예규문 자체에 "사실관계가 다르면 답변이 달라질 수 있다"는 면책 단서가 붙는다.
반면 **세법해석 사전답변**은 과세관청을 구속한다(신뢰보호). → 출하할 때 구속력 등급을
같이 주지 않으면 소비 LLM 이 예규를 법조문처럼 인용한다. **F20 과 같은 종류의 함정이다.**

세법 기본통칙은 `admrul` 에서 0건이다("소득세법기본통칙"·"법인세법 기본통칙"·"기본통칙" 전부) —
이것만은 법제처 경로가 없다.

## 5. 이 프로브가 뒤집는 기존 판단

1. **조세심판원 "미확인" → 실재 확인.** 선행 조사의 어순 오류였다. 세법 vertical 의 핵심 재료가
   비용 낮게 열려 있다.
2. **`decc`(행정심판재결례)로는 조세를 못 잡는다.** `target=decc&query=가산세` 는 totalCnt 0 이다 —
   조세심판원은 일반 행정심판과 **별도 컨테이너**다. 현재 MCP 의 `source` enum 14종에 조세가
   빠져 있는 이유가 이것이다.
3. **`type=HTML` 폴백을 조세심판원에 쓰면 안 된다.** IB1b 에서 NTS 판례에 HTML 폴백을 붙인 선례가
   있어 같은 처방을 재사용하기 쉬운데, 여기서는 HTML 이 빈 껍데기다.
4. **별표는 "찾기"와 "읽기"가 다른 문제다.** 선행 조사는 `licbyl` 을 한 줄로 4순위에 뒀는데,
   실제로는 메타데이터(싸다)와 파일 본문(비싸다)이 갈린다.
5. **"이 OC 로는 빈 응답" 판정 3건이 전부 어순 오류였다.** `specialDeccTt`→`ttSpecialDecc`,
   `cgmExpc{코드}`→`ntsCgmExpc`. 법제처 target 명명 규칙은 **`{기관약어}{서비스}`** 이고,
   공식 문서 표기가 이를 뒤집어 적은 것으로 보인다. 남은 빈 응답 target(`lsRlt`·`couseLs`·
   `drlaw` 등)도 같은 의심을 받아야 한다 — 이번 horizon 범위 밖이지만 백로그에 남긴다.
6. **시점 조회의 함정이 무경고다.** `target=law&efYd=…` 가 인자를 삼키고 현행을 준다.
   과세연도 축을 다루는 순간 이건 오답 생성기다.

## 6. 이 프로브가 닫지 못한 것 (해당 milestone 의 첫 검증 항목)

- `ntsCgmExpc` **전문 조회 계약** — 단건객체 결함으로 ID 추출 실패. 검색 메타만 확인됨.
- `licbyl` 별표 **파일 본문 추출** 가능 여부(HWP/PDF) — 링크 존재만 확인.
- `ttSpecialDecc` 데이터 기준일 2024.12.18 / `ntsCgmExpc` 2025.06.28 — **최신 자료는 없다.**
  "최근 결정례" 질의에는 구조적으로 답 못 한다. 한계로 선언하고 경고에 실어야 한다.
