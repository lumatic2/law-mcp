## 1. 세무 실무자는 무엇을 위계적으로 참조하는가

**[확인] 위계 자체는 실무 블로그·법률 자료에서 일관되게 다음 순서로 서술된다:**
법령(법률) → 시행령 → 시행규칙 → 기본통칙/집행기준(국세청 행정규칙) → 예규·유권해석(서면질의·사전답변·질의회신) → 조세심판원 결정례 → 법원 판례.
출처: [(법,시행령,시행규칙,통칙,집행기준,예규,판례,서면질의 등)](https://blog.naver.com/jangsemoo_/223787388908) (접근 2026-07-21), [민원 문서/판례/행정규칙/세법 집행기준](https://blog.naver.com/gnbab/224151895739) (접근 2026-07-21).

**[확인] 법적 구속력은 위계와 반대로, 위로 갈수록 강하고 예규·통칙은 "구속력 없는 자료"에 불과하다** — 대법원 판례 인용:
- "법원이 세법을 해석함에 있어 이를 하나의 자료로 사용할 수 있다" (대법원 1994.12.22 판시로 인용됨)
- "행정청 내부를 규율하는 규정일 뿐 국가와 국민 사이에 효력을 가지는 법규가 아니므로, 법원이나 일반 개인에 대한 법적 구속력은 없다" (대법원 1998.9.8 판시로 인용됨)
- 반면 **세법해석 사전답변**은 "공적견해 표명으로 당해 질의에 대해 과세관청을 구속"(신뢰보호원칙 적용) — 서면질의보다 구속력이 강한 별도 트랙.
출처: [조세불복전문세무사 — 서면질의·사전답변·예규·유권해석·집행기준·기본통칙의 법적 구속력과 법원성](https://taxly.kr/post/780-...) (접근 2026-07-21).

**[확인] 예규는 실무에서 "참고"로 광범위하게 인용되지만, 그 예규문 자체에 면책 단서가 붙는 경우가 흔하다:**
"국세청 예규의 마지막 문단에는 실무적으로 가장 무서운 단서 조항이 달려 있다. '다만, 이 질의가 사실관계와 다를 경우에는 답변 내용이 달라질 수 있다'" — 세무사 실무 기술.
출처: [하도급 거래 분쟁 합의금, 법인세 손금 인정될까?_국세청예규](https://blog.naver.com/kkksilverz/224347126767) (접근 2026-07-21).

**[확인] 조세심판원 결정례·법원 판례는 국세청이 직접 운영하는 국세법령정보시스템(taxlaw.nts.go.kr)에서 함께 공개되어 실무자가 상시 조회한다.**
출처: [조세불복(2) — 국세법령정보시스템](https://blog.naver.com/yeonwoolaw/223307504182) (접근 2026-07-21), [taxlaw.nts.go.kr 전체 판례·결정례](https://taxlaw.nts.go.kr/pd/USEPDI001M.do) (접근 2026-07-21).

**"예규 행정"이라는 관행 — [확인 부분] / [미확인 부분]**
- [확인] "예규를 근거로 과세관청이 사실상 법령처럼 운용하는 관행"에 대한 문제의식은 실재한다 — "국세청 과세품질 높이기 위해선 법령정보시스템상 예규 정비부터 신속히" 같은 국세행정포럼 기사, 실무자가 예규 변경·정합성 이슈를 반복 지적.
  출처: [taxtimes — 국세행정포럼 기사](http://www.taxtimes.co.kr/news/article.html?no=261186) (접근 2026-07-21).
- [미확인] 다만 "예규 행정"이라는 **고정된 용어 자체**가 학계·실무에서 표준 명칭으로 정착했는지는 이번 조사로 확증하지 못했다. 검색된 자료는 전부 "예규", "유권해석", "행정해석" 등 개별 용어로 서술했고, "예규 행정"을 하나의 복합명사로 쓰는 출처는 발견하지 못했다. 존재하지 않는다는 뜻은 아니며, 추가 검색어(예: "예규행정" 붙여쓰기, 학술논문 DB)로 재확인이 필요하다.

**평가 세트 분포 제안 [판단, 근거는 위 표]**: 비과세/공제 요건 판단(양도세·법인세 손금·세액공제)이 실무 질의의 다수를 차지하는 것으로 관찰되므로, 이 축을 40% 내외로 두고 상속·증여, 불복 절차, 원천징수/가산세(법령 기반 보강), 사전질의 제도 이해를 나머지에 분산하는 구성이 실무 대표성에 부합할 것으로 보인다.

## 3. 시간축(과세연도)이 결정적인 이유

**[확인] 개정이 잦은 대표 조문 = 조세특례제한법(조특법)**. 검색된 법제처 입법예고·법령 이력 페이지 자체가 이를 뒷받침한다:
- [법제처 — 조세특례제한법 일부개정법률(안) 입법예고](https://www.moleg.go.kr/lawinfo/makingInfo.mo?mid=a10104010000&lawSeq=79300&lawCd=2000000308488&lawType=TYPE5&currentPage=1) (접근 2026-07-21)
- 국가법령정보센터의 조특법·조특법 시행령 페이지는 개정 이력이 방대하다는 점이 별도로 통상 알려져 있음 [확인, 국가법령정보센터 페이지 자체에서 개정 연혁 확인 가능]: [law.go.kr 조세특례제한법](https://www.law.go.kr/%EB%B2%95%EB%A0%B9/%EC%A1%B0%EC%84%B8%ED%8A%B9%EB%A1%80%EC%A0%9C%ED%95%9C%EB%B2%95) (접근 2026-07-21)

**[확인, 실무 사례] 귀속연도에 따라 적용 법령이 갈리는 실제 사례**:
- 이월과세 강화(증여 후 양도 시 적용 이월과세 기간): "2026년부터 특히 주의해야 할 리스크 — 증여 후 양도 시 '이월과세' 강화: 부모님으로부터 증여받은 주택을 10년 이내 매도할 경우" — 연도별 개정으로 이월과세 적용기간(5년→10년 등)이 달라진 사례.
  출처: [2026년 1가구 1주택 양도소득세 비과세 요건 정리](https://blog.naver.com/tigersblog/224130949284) (접근 2026-07-21)
- 조세심판원 결정례 자체가 "귀속년도"를 사건의 1급 메타데이터로 관리한다: "조세심판원-2014-중-0424 / 귀속년도: 2007 / 심급: 심판" — 즉 심판원도 어느 연도 법령을 적용했는지를 사건 식별의 핵심 축으로 삼는다.
  출처: [blog.naver.com/pack3031203](https://blog.naver.com/pack3031203/223600961134) (접근 2026-07-21)

**[판단, 근거는 위]** 이 두 사례는 "몇 년 귀속인가"가 세법에서 조문 검색만큼 중요한 축임을 보여준다 — 같은 조문 번호라도 시행 연도에 따라 요건(거주기간·비과세 한도·이월과세 기간)이 다르므로, MCP가 조문만 반환하고 시행일자를 명시하지 않으면 오답 위험이 구조적으로 존재한다.

## 5. 국세청 예규·조세심판원 결정례의 프로그램적 접근 경로

**[확인] 법제처 OPEN API에 관련 target이 문서상 존재한다:**

- **`ntsCgmExpc`** — 국세청 법령해석(예규) 목록 조회. 요청 URL: `http://www.law.go.kr/DRF/lawSearch.do?OC={인증키}&target=ntsCgmExpc&type=XML|JSON|HTML&query=...`. 파라미터: `query`(검색어), `search`(1=제목/2=본문), `display`(최대100), `page`, `sort`, `explYd`(해석일자 범위), `itmno`(안건번호). 응답 필드: 안건명·안건번호·질의기관명·해석기관명·해석일자·법령해석일련번호.
  출처: [open.law.go.kr — cgmExpcNtsListGuide](https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=cgmExpcNtsListGuide) (접근 2026-07-21), [data.go.kr — 법제처_국세청 법령해석 목록 조회](https://www.data.go.kr/data/15140313/openapi.do?recommendDataYn=Y) (접근 2026-07-21)
- **`expc`** — 법령해석례 목록 조회(범용, 기관 특정 없음). `target=expc`. [open.law.go.kr — expcListGuide](https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=expcListGuide) (접근 2026-07-21)
- **`specialDeccTt`** — 조세심판원 특별행정심판례. 공식 가이드 목록 페이지에 항목명은 존재하나(`open.law.go.kr/LSO/openApi/guideList.do`), 이 레포가 실제로 보유한 OC 키로 `lawSearch.do?target=specialDeccTt&query=...` 형태 호출을 시도한 결과 **빈 응답**이었다 — 이는 이 레포의 이전 리서치(`research/2026-07-21-lawgo-api-survey.md` §D-2, 접근 2026-07-21 작성)에서 이미 실측·기록됨. "서비스가 없다"가 아니라 "이 호출 형태·이 OC 권한으로는 접근 안 됨"으로 해석해야 한다는 단서가 같은 문서에 있음.

**[미확인]** `specialDeccTt`가 정확히 어떤 추가 파라미터(별도 신청·권한 등급·다른 엔드포인트 `lawService.do` 사용 여부)를 요구하는지는 이번 조사로도 확정하지 못했다. WebFetch로 접근한 `guideList.do` 페이지는 JS 렌더링 특성상 조세심판원 항목의 실제 링크(htmlName 값)를 텍스트로 노출하지 않았다 — 브라우저 렌더링(Playwright 등)으로 재확인이 필요하다.

**[미확인] 국세법령정보시스템(taxlaw.nts.go.kr) 자체의 공개 API 여부** — 이번 조사에서 웹 UI 검색 페이지(`USEQTJ001M.do`, `USEPDI001M.do`)만 확인했고, 국세청이 별도로 REST API를 공개한다는 근거는 찾지 못했다. `data.go.kr`(공공데이터포털)에 등록된 API는 법제처가 수집·재공개하는 `ntsCgmExpc` 계열이며, 국세청 자체 API로 보이지 않는다.

**[미확인] 조세심판원(tt.go.kr) 자체 API** — tt.go.kr 사이트 존재는 확인했으나([tt.go.kr](https://www.tt.go.kr/), 접근 2026-07-21), 사이트 자체가 API를 공개하는지는 확인하지 못했다. 법제처 경유(`specialDeccTt`)가 유일한 프로그램적 경로일 가능성이 높으나 위에서 보듯 현재 OC로는 미작동.

**결론 [판단]**: 예규(국세청 법령해석)는 `ntsCgmExpc`로 인증키만 있으면 즉시 접근 가능해 보인다(파라미터 형태까지 확인됨 — 실측 호출은 아직 안 함, 문서 기준). 조세심판원 결정례는 문서상 target은 있으나 이 레포 환경에서 실제로 뚫리지 않은 상태이며, 이것이 이번 horizon의 첫 번째 실험적 스파이크(spike) 대상이 되어야 한다.
