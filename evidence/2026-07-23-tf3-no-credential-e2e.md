# TF3 무자격 실표면 관측

> 2026-07-23 · milestone TF3 step-3 · horizon `trap-free`

## 무엇을 판정하나

**우리는 이미 인증값을 갖고 있어서 이 함정을 못 본다.** 문서 diff 는 증거가 아니다 —
인증값이 없는 사용자가 실제 MCP 클라이언트에서 받는 문자열이 증거다.

`.env` 는 건드리지 않았다(시크릿 파일). 자식 프로세스에 `LAW_API_OC=""` 를 **명시로** 넘겼다 —
dotenv 는 이미 정의된 키를 덮어쓰지 않으므로 디스크에 `.env` 가 있어도 빈 값이 유지된다.
재현: `npx tsx src/no-credential-smoke.ts`

## ① 인증값 없음 — 클라이언트가 받은 원문

```
=== 서버가 기동은 되는가 ===
연결 성공. instructions 있음

=== search_law (상류 조회 — 실패해야 정상) ===
ERROR: 법제처 OpenAPI 인증값(LAW_API_OC)이 없습니다 — 이 서버의 도구 11개 중 9개가 동작하지 않습니다.

1. 발급 신청: https://open.law.go.kr/LSO/openApi/cuAskList.do (메일주소로 회원가입·로그인한 뒤 신청)
2. 발급된 값 확인: https://open.law.go.kr/LSO/usr/usrOcInfoMod.do
3. 그 값을 .env 의 LAW_API_OC 에 넣거나, MCP 설정의 env 로 전달하세요.

인증값은 사용자가 직접 발급받아야 하며 이 서버가 대신 발급할 수 없습니다.

=== batch_validate_legal_terms (로컬 규칙 — 인증값 없이도 돌아야 정상) ===
{ "items": [ { "term": "절세팁", "status": "ok", ... } ], "warnings": [] }
```

- 서버는 **죽지 않고 기동**한다. 인증이 필요한 도구만 안내를 주고, 로컬 도구는 돈다.
- 스택트레이스·내부 경로 노출 **없음**(실패 검증 통과 조건).

## ② 잘못된 인증값 — 실패 검증이 실제 결함을 잡았다

```
=== search_law ===
ERROR: 법제처 API 일시 장애: HTTP 500
이 오류가 반복되면 일시 장애가 아니라 인증값 문제일 수 있습니다 — 위 인증키관리에서 값과
IP·도메인 등록 상태를 확인하세요.

=== get_law_article ===
ERROR: 법제처 API 인증 실패: 사용자 정보 검증에 실패하였습니다. OPEN API 호출 시 사용자
검증을 위하여 정확한 서버장비의 IP주소 및 도메인주소를 등록해 주세요.
확인할 것: ...
```

**결함**: 상류가 잘못된 인증값에 두 가지로 답한다 — 어떤 경로는 HTTP 200 + 오류 본문,
어떤 경로는 5xx. 후자는 우리 분류기에서 "일시 장애(재시도 가능)"가 되어, **인증 문제인
사용자가 영원히 재시도하게 된다.** 상류 분류를 우리가 고칠 수 없으므로 반복 시 무엇을
의심해야 하는지를 메시지에 남겼다. 근본 수리(5xx 시 검증용 재조회로 재분류)는 후속 후보.

상류 원문 확인:

```
$ curl 'https://www.law.go.kr/DRF/lawSearch.do?OC=<잘못된값>&target=law&type=JSON&query=소득세'
status 200
{ "result": "사용자 정보 검증에 실패하였습니다.",
  "msg": "OPEN API 호출 시 사용자 검증을 위하여 정확한 서버장비의 IP주소 및 도메인주소를 등록해 주세요." }
```

→ **IP·도메인 등록이 실제로 필요하다.** 공개 문서 3곳에는 없던 사실이라 앞서 README 에 쓰지
못했는데(추정 금지), 상류가 직접 말해 줘서 근거와 함께 반영했다.

## ③ 정상 인증값 회귀 없음

`npm run smoke:mcp` — `search_law` 정상 응답(법령 6건 조회 확인). 진단 문구 추가가 정상 경로를
막지 않았다.

## ④ 배포 사본

```
$ cd ~/projects/custom-mcps/law-mcp && git pull && npm run build
$ grep -c "법제처 OpenAPI 인증값" dist/config.js   → 1
$ npx tsx src/no-credential-smoke.ts               → 같은 안내 출력
```

배포 사본 `2c668b9` 까지 반영, dist 스모크 통과.

## ⚠ 재시작 부채

**MCP 서버 재시작이 필요합니다(사용자).** 배포 사본 빌드는 끝났지만 현재 세션에 붙어 있는
law-mcp 프로세스는 여전히 구 `dist` 를 실행 중입니다. 재시작 전까지 이 세션의 도구 응답에는
새 진단 문구가 나오지 않습니다.

## 평가 못 함

- **IP·도메인 등록을 실제로 해 보지는 않았다.** 상류 문구로 필요성을 확인했을 뿐이다 —
  등록 화면의 동작·소요 시간은 미확인.
- 5xx 오분류의 근본 수리(재분류 로직)는 하지 않았다. 메시지 보완까지만.
