# 행정 도메인 결함 진단 — 순위 문제가 아니라 **도달 불가** 문제였다

- 일자: 2026-07-21 (horizon `upstream-delivery` close 직후)
- 계기: 홀드아웃 개봉에서 유일하게 틀린 1건이 dev 의 약점과 같은 자리였다 (행정 60~67%)
- 원자료: `2026-07-21-body-pool-ab.json` (교차 A/B) · 아래 실 API 프로브
- **판정: 이 결함은 지금 고칠 수 없다. 채택한 변경 없음.** 다음 milestone 의 입력으로 남긴다.

## 실패 3건은 전부 같은 유형 — 행정 총칙법

| split | 쿼리 | 정답 | 정답 법의 조문제목 |
|---|---|---|---|
| dev | 처분 사전통지 의무 | 행정절차법 | 제21조 **처분의 사전 통지** |
| dev | 과징금 부과 기준 | 행정기본법 | 제28조 **과징금의 기준** |
| holdout | 이행강제금 부과 | 행정기본법 | 제31조 **이행강제금의 부과** |

셋 다 **정답 법의 조문제목이 쿼리와 거의 정확히 일치**한다. 그런데도 못 찾는다.

## upstream 의 어느 채널도 정답을 주지 않는다 (`이행강제금 부과` 실측)

| 채널 | 결과 | 행정기본법 |
|---|---|---|
| `aiSearch` (UD2 채택 채널) | 20위까지 조회 | **없음** |
| `aiRltLs` (F6 후보) | 10건 전부 | **없음** — 은행법·자본시장법·금융회사지배구조법 |
| 용어 연계 `lstrmAI` | `totalLinks: 0` | **없음** (`이행강제금` 은 법령용어가 아니다) |
| 본문검색 `search=2` | 120건 중 **116번째** | 있으나 꼬리 |

`aiRltLs` 는 우리 파이프라인과 **똑같은 오답**(은행법 1위)을 준다. `lsRlt` 는 조사 시점부터 빈 응답이다.

> **F6(`lsRlt`/`aiRltLs` 관계 그래프)은 이 결함의 재료가 아니다.** 백로그에서 이 용도로는 내린다.

## 구조적 결함 — 본문검색은 가나다순이고 우리는 앞부분만 본다

본문검색은 관련도순이 아니라 **가나다순**이다(LB2 부터 알던 사실). 그런데 우리가 받는 건
`display = max(limit*3, 30)` — **앞 30건**뿐이다. 정답이 뒷글자로 시작하면 재정렬로 닿을 수 없다.

```
"이행강제금 부과"  totalCnt=120
  받던 것: 가맹사업거래…~공직자의 이해충돌 방지법 (앞 30건)
  행정기본법(ㅎ): 116번째 — 후보 풀에 아예 없었다
```

**순위 문제가 아니라 후보 풀 문제였다.** `행정`·`환경`·`형사`·`한국…` 처럼 뒷글자로 시작하는
법령은 본문검색 경로에서 구조적으로 탈락한다.

## 고쳐 봤고, 채택하지 않았다

풀 전체(상한 300, 페이지 병렬)를 받아 정렬하도록 고쳐 교차 A/B 를 돌렸다:

```
본문검색 첫 페이지만 (구 동작)   88.0% (22/25)
본문검색 풀 전체                88.0% (22/25)
  이득 0 / 손실 0 / 순 +0  → 채택 불가 (F13 게이트)
```

**손실은 0 이지만 이득도 0 이다.** 이유는 상한이다:

| 쿼리 | totalCnt | 상한 300 으로 도달? |
|---|---|---|
| 이행강제금 부과 | 120 | ✅ 풀에 들어옴 (그래도 순위 밖) |
| 과징금 부과 기준 | **613** | ❌ 앞 300 은 ㄱ~ㅅ, 행정기본법 못 옴 |
| 처분 사전통지 의무 | **608** | ❌ 행정절차법 못 옴 |

전부 받으려면 613건 = **7 페이지**다. UD2 에서 이미 상류 쓰로틀로 한 번 멈춘 적이 있는데
검색 1회에 7 호출을 얹는 건 지연·차단 위험이 이득보다 크다. 그래서 **되돌렸다**(코드 미채택).

## 남은 길 두 개 (다음 milestone 의 입력)

1. **조문제목 신호를 만든다.** 정답 3건 모두 조문제목이 쿼리와 일치한다 — 이건 이름 길이보다
   훨씬 강한 관련도 신호다. 재료(`src/article-index.ts` `extractArticles`)는 LB2 에 이미 있다.
   문제는 *어느 후보의* 조문을 받아 볼지다(법령 전문 1건이 300~600KB). 후보를 좁히는 규칙이 필요하다.
   - 이름 형태 규칙 후보: `~기본법`·`~절차법` = 총칙 계열. 풀에서 **2~5개**로 좁혀진다(실측).
     단 `국세기본법`·`건설산업기본법` 이 함께 걸리므로 **조문제목 검증이 필수**다(승격 전 확인).
   - ⚠ 이 규칙은 dev 2건을 보고 만드는 것이라 **F5 과적합 위험이 정확히 그 자리**다.
     홀드아웃 15건은 close 에서 소진됐다 — **새 평가 세트 없이 이 규칙을 채택하면 안 된다.**
2. **총칙-개별법 관계를 upstream 에서 얻는다.** 현재 확인된 채널에는 없다. `lsStmd`(법령체계도)·
   `thdCmp` 는 아직 미검증이라 프로브 가치가 있다.

## 이 조사가 확정한 것

- 행정 도메인 결함은 **랭킹 튜닝으로 못 고친다** — 정답이 후보 풀에 없거나 신호가 없다.
- **F6 은 재료가 아니다** (`aiRltLs` 가 같은 오답, `lsRlt` 빈 응답).
- 본문검색 가나다순 절단은 **실재하는 구조적 결함**이다. 다만 단독으로는 이득이 0 이라
  조문제목 신호와 **함께** 채택해야 한다.
- 따라서 이 건은 step-grade 가 아니라 **milestone-grade**다 (새 평가 세트 + 신호 설계 + A/B).
