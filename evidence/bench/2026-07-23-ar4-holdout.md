## 홀드아웃 실측 — 그리고 왜 이 숫자를 믿으면 안 되는가

| | 홀드아웃 10건 × 5회 |
|---|---|
| `pass^5` (5회 전부) | **100%** |
| `SR@1` (첫 질의) | **98%** |
| `AT` | 1.02 |
| 침묵률 | 0% |

앞 3회만 뽑으면 `pass^3` **100%** · `SR@1` **100%** — dev(`pass^3` 90% · `SR@1` 80%)보다
**높다**. 과적합은 기각되지만, 이 세트는 **변별력이 거의 없다.**

**⚠ 원인은 세트 제작 방식이다(프리모템 ① 발화).**

| | 질의 주제 출처 | 맥락 문단 작성 |
|---|---|---|
| dev 20건 | **20/20 구 세트**(단발 벤치용으로 독립 작성) | 나 |
| 홀드아웃 10건 | **6/10 내가 새로 고안** | 나 |

홀드아웃은 **주제까지 내가 골랐고**, 고를 때 도구로 조문 존재를 확인했다. 즉 **도구가 잘
다루는 영역에서 문제를 낸 것**이다. 유출 탐지기는 문자열만 보므로 이런 **구성에 의한 오염**을
못 잡는다. horizon 프리모템 ①("맥락 문단에 답을 흘려 변별력 0 인 자를 만든다")이 문자열이
아니라 **주제 선택 경로로** 발화했다.

**따라서 홀드아웃 100% 는 "도구가 완벽하다"가 아니라 "내가 쉬운 문제를 냈다"로 읽어야 한다.**
과적합 기각이라는 결론만 살리고, 절대 수치는 신뢰하지 않는다.

## AR3 재판정 — `SR@1` 순 이득은 **미달**

AR3(어휘 공백 경고)를 "손실 0 이 구조적"만으로 채택하고 `SR@1` 순 이득은 미실측으로 남겼다.
`--no-vocab-gap` 대조군 블라인드 3회로 갚았다:

| | 경고 ON (제품) | 경고 OFF (대조군) | 차 |
|---|---|---|---|
| `SR@1` | 80.0% | **88.3%** | **−8.3%p** |
| `pass@3` | **100%** | 95% | +5%p |
| `pass^3` | 90% | 90% | 0 |
| `AT` | 1.16 | 1.05 | +0.11 |

**AR3 의 선언 문턱("손실 0 AND `SR@1` 순 이득 ≥2")은 충족되지 않았다.** `SR@1` 은 오히려
낮다. 기전이 설명된다 — 경고의 **정밀도가 75%** 라, 이미 성공한 검색 넷 중 하나에 대고
"다시 물어라"라고 말한다. 그러면 첫 질의로 끝났을 것이 재질의로 밀린다.

동시에 `pass@3` 는 올랐다(95% → 100%). **한 번이라도 도달할 확률은 올리고, 첫 질의로 끝낼
확률은 내린다.** 어느 쪽이 나은지는 제품 판단이지 측정이 정하는 게 아니다.

⚠ **표본이 작다.** 60 실행 중 5건 차이라 통계적으로 확정된 열세가 아니다. 방향만 읽는다.

## 실 MCP 표면 E2E (닫는 기준 ④ 대비책)

**재시작된 실 MCP 서버**에서 관측했다(배포 dist 대체 실행 아님 — 서버 `instructions` 가 이
세션에 실제로 실려 왔다).

```
1턴  "세금을 잘못 매겼을 때 다투는 절차"
     → 부가가치세법·세무사법. 정답 없음
     ⚠ "질의 어휘가 결과에 거의 나타나지 않는다(0/5 토큰) … 법률 용어로 바꿔 다시 검색하라"
2턴  "위법한 처분에 대한 심사청구 심판청구 불복"  → 국세기본법 §55 「불복」 ✅
3턴  as_of=2024  → 본문 + effective_date 20240401 + as_of_rule(귀속연도 해석)
```

**결함 1건 발견 — `as_of` 가 법령ID 로는 실패한다.**
`get_law_article(law_id="001586", …, as_of="2024")` → 에러. `law_id="국세기본법"` 은 정상.
`search_law` 는 ID 를 주는데 `as_of` 는 이름을 요구하므로 **검색 결과를 그대로 물려주면 끊긴다.**
도구가 조용히 현행을 주지 않고 에러를 내는 것은 옳다(틀린 연도는 오답). 문제는 경로 부재다.
AR4 범위(판정만, `src/` 불변)라 **finding 큐**로 넘긴다.
