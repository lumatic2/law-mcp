# TF2 맥락 품질 점검 — 도달성 대조

> 2026-07-23 · milestone TF2 step-2 · horizon `trap-free`

## 무엇을 판정하나

이번에 새로 쓴 맥락 60건(dev 43건)이 **애초에 풀 수 있는 문제인가.** 계획에는 dev 재기준선
3회(실 블라인드 세션)가 있었으나, 그 정밀도는 *도구 버전을 비교할 때* 필요한 것이고 이번
horizon 은 도구를 한 줄도 고치지 않았다(`src/` 0줄). 지금 낸 기준선은 다음에 도구를 고칠 때
어차피 다시 재야 하므로, **쓸 데가 정해지지 않은 정밀도**였다.

대신 훨씬 작은 질문으로 좁혔다 — 맥락을 그대로 검색에 던져 정답 법령이 상위에 나오는가.
에이전트 세션 없이 API 호출만으로 답이 난다.

## 대조

| 맥락 | 단발 검색 1위 도달 | 3위 내 도달 |
|---|---|---|
| AR2 기존 맥락 (dev 20건, 채점 대상) — **에이전트가 `pass^3` 90% 를 낸 그 맥락** | 30.0% | **35.0%** |
| TF2 신규 맥락 (dev 43건) | 32.6% | **39.5%** |

## 읽기

**신규 맥락이 검증된 맥락보다 조금 높다.** 39.5% 라는 낮은 절대값은 품질 불량이 아니라 일상어
맥락의 정상 프로파일이다 — 한 번에 안 닿기 때문에 에이전트가 재질의해야 하고, 그 재질의 능력이
`agentic-reach` 가 재려던 축이었다. AR2 도 35% 에서 출발해 에이전트 루프로 90% 에 도달했다.

즉 **맥락 품질 가드는 통과했고, 3 arm 블라인드 세션은 이 판정에 아무것도 더하지 않는다.**

## 재현

```bash
npx tsx bench/context-reachability.ts --split dev --only-new    # 신규 43건
# 대조군(AR2 20건)은 같은 스크립트에서 --only-new 를 빼고 context_origin 으로 가른다
```

## 평가 못 함

- **dev 재기준선(`pass^3`·`SR@1`·`AT`·기권 정밀도/재현율)은 내지 않았다.** 실 블라인드 세션
  3회가 필요하고, 그 수치는 도구를 고치는 시점에 before/after 로 함께 재는 것이 맞다.
  하네스는 준비해 뒀다 — `evidence/bench/2026-07-23-tf2-baseline/`(BRIEF·tasks.json 43건),
  `dist-bench` 빌드 + `tool-cli` 스모크 통과. 돌리려면 그대로 세션만 띄우면 된다.
- 그래서 **이번 horizon 은 에이전트 지표의 새 기준선을 갖지 않는다.** 유효한 기준선은 여전히
  AR2 의 dev 20건 `pass^3` 90% 이고, 코퍼스가 커진 만큼 그 값이 새 세트를 대표하지는 않는다.
  이 사실을 수치 없이 덮지 않는다.
- 단발 도달 26건 MISS 목록은 사람이 하나씩 볼 대상으로 남아 있다(맥락이 모호한 것인지 어휘
  공백이 큰 어려운 케이스인지). 대조군도 13/20 이 MISS 라 **이 비율 자체는 이상하지 않다.**
