# 최종 보고서 — UD2 `aiSearch` 편입 + A/B 판정

> 완료: 2026-07-21 · 대상: UD2 (horizon `upstream-delivery`) · 작성: 완료 경계(§B3)

## 1. 문제 정의

이 레포는 다섯 milestone 동안 **"upstream 에 관련도 랭킹이 없다"**를 전제로 후보 생성기를 직접
만들었다. 본문검색이 가나다순이라는 실측이 그 전제를 받쳐 줬다. 전수 조사(2026-07-21)에서
전제가 틀렸음이 드러났다 — 법제처는 자연어 질의를 **조문 단위로** 답하는 `aiSearch` 를 이미
제공하고 있었고, 무튜닝으로 우리 튜닝을 이겼다.

## 2. Objective 연결

**정확도** 축을 정면으로 밀었다. 1위 정확도 32% → 72% 는 소비 LLM 이 목록을 고르지 않아도
된다는 뜻이고, 그게 이 도구가 에이전트 안에서 쓸 만해지는 조건이다.
**함정 없음** 축도 함께 — 조문을 출처와 함께 주므로 "어느 법인지"까지만 받고 나머지를
추론하다 틀리는 경로가 줄었다.

## 3. 경로

horizon `upstream-delivery` 의 두 번째 milestone. step-1(클라이언트)만 끝난 채 **F12(상류
쓰로틀 의심)로 멈춰 있었다.** 사용자 지시로 UD3 를 먼저 완주하고, 휴지 뒤 이 milestone 을 재개했다.

계획한 4 step 을 순서대로 밟았다. 계획과 다른 점 하나: step-2 에서 A/B 가 채택 조건을 넘자
**기본값을 그 자리에서 켰다**(계획은 step-4). 판정이 이미 섰는데 미루면 step-3 이 꺼진 경로를
측정하게 된다.

## 4. 구현 결과

**주제 검색이 법제처의 관련도 순위를 1순위 신호로 쓴다.** dev 25건 정답 포함률 48% → 80%,
1위 정확도 32% → 72%. 어느 도메인도 내려가지 않았다(행정 20→60이 최대 상승).

**"어느 법"에서 멈추지 않고 "몇 조"까지 준다.** `행정심판 청구 기간` → 행정심판법 **제27조
(심판청구의 기간)**. upstream 이 이미 조문 단위로 답하므로 **추가 호출 0**이다.

**기존 사다리는 하나도 지우지 않았다.** `aiSearch` 는 앞단 채널일 뿐이라 편입 지점 1곳을 끄면
LB5 상태로 즉시 복귀한다. 도구 개수도 불변(11개).

## 5. 이슈와 해결

**이슈 1 — "대체가 아니라 병합"을 값으로 확인했다.**
`aiSearch` 단독은 recall@3 이 병합과 **똑같이 80%** 다. 집계만 보면 구분되지 않는다. 쿼리
단위로 보니 갈렸다 — `사용자책임 면책 사유` 를 기존 사다리는 맞히는데 `aiSearch` 는 놓친다.
**단독 채택이었으면 이게 회귀로 출하됐다.** 집계 지표만 봤으면 못 잡았을 판정이다.

**이슈 2 — 테스트가 실 API 를 타고 있었다(적발).**
기본값을 켜는 순간 `term-boost.test.ts` 4건이 깨졌다. 원인은 그 테스트들이 `aiSearch` 를
주입하지 않아 **진짜 법제처를 호출**하고 있었다는 것. 파일 상단에 "실 API 를 타지 않도록
주입한다"고 적혀 있었는데 새 채널이 그 격리를 뚫었다. 주입 격리를 복구했다.

**이슈 3 — 지연 DoD 를 한 번 놓쳤다.**
첫 측정 중앙 4079ms(DoD ≤3000ms). 사다리와 `aiSearch` 를 **순차**로 돌린 탓인데, 두 채널은
서로의 결과를 안 쓴다. 동시 실행으로 바꿔 2939ms — 대조군 2958ms 와 사실상 같다.
**채널을 하나 더 붙이고도 느려지지 않았다.**

**이슈 4 — F12 는 해소됐다.** 휴지 후 재측정 48.0% ×3(σ=0). 60% 로 돌아오지 않았으므로 48% 가
정상 기준선이다. "쓰로틀이 없다"의 증명은 아니고 회복이 없었을 뿐이라, 판정은 여전히 절대
수치가 아니라 교차 측정으로 했다.

## 7. 후속 제안

- **horizon close 판단이 다음이다** — UD1·UD2·UD3 가 모두 닫혔다. 닫는 기준 5개 중 조문 출하
  (≥60% → 76%)·판정 가능성·자료원·회귀는 충족했고, **recall@3 ≥85%(현재 80%)** 만 미달이다.
  홀드아웃 개봉은 horizon close 시 1회뿐이라 그 판단이 곧 세트의 운명이다.
- **F19**(신규): 검색 지연 꼬리가 5.4초까지 간다. `aiSearch` 가 아니라 **기존 사다리**의 꼬리다
  (대조군이 5.6초로 더 길다). `aiSearch` 가 답한 경우 사다리 하위 단계를 생략할 수 있다.
- **F6**: `aiRltLs`(AI 연관법령)·`lsRlt`(관련법령 그래프)는 같은 계열의 미사용 API.
- 못 맞히는 4건(육아휴직 신청 요건과 기간 · 범죄수익 몰수 추징 · 처분 사전통지 의무 ·
  과징금 부과 기준)은 상류가 못 주는 것이라 랭킹으로 안 풀린다.
- **MCP 재시작 필요** — `search_law` description 과 응답 필드(`ai_articles`)가 바뀌었다.
