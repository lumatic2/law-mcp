- **README 가 도구 4/11 → 11/11.** 판례·행정규칙·해석자료·`verify_citation`·`as_of` 가 문서상

- **OC 발급 안내** — 법제처 신청·인증키관리 링크 + 접근일. 화면 단계는 복제하지 않는다(상류가
  바뀌면 낡는다). 인증값 없이는 11개 중 9개가 안 돈다는 사실을 앞세웠다.
- **무자격·인증 실패가 사람이 읽는 진단**이 됐다. 구 문구는 `LAW_API_OC is missing. Set it in

- 실 MCP 클라이언트에서 무자격 상태를 관측해 원문을 증거로 남겼다. 서버는 죽지 않고 기동하며
  스택트레이스·내부 경로 노출이 없다.

## 2. 이슈와 해결

- **실패 검증이 실제 결함을 잡았다.** 잘못된 인증값에 상류가 경로별로 다르게 답한다 — 어떤
  경로는 HTTP 200 + 오류 본문, 어떤 경로는 5xx. 후자는 우리 분류기에서 "일시 장애(재시도 가능)"가
  되어 **인증 문제인 사용자가 영원히 재시도한다.** 상류 분류를 고칠 수 없으므로 반복 시 무엇을
  의심할지 메시지에 남겼다. 근본 재분류는 후속 후보(F1).
- **앞서 못 썼던 사실을 확보했다.** IP·도메인 등록 필요 여부는 공개 문서 3곳에 없어 추정 금지
  원칙상 README 에 못 썼는데, 상류가 오류 응답에서 직접 말해 줬다("정확한 서버장비의 IP주소 및
  도메인주소를 등록해 주세요"). 근거와 함께 반영했다.
- **`.env` 를 건드리지 않고 무자격을 만들었다.** 시크릿 파일 이동은 위험하고 가드도 막는다.
  자식 프로세스에 `LAW_API_OC=""` 를 명시로 넘기는 방식으로 우회했다(dotenv 는 이미 정의된 키를
  덮어쓰지 않는다).
