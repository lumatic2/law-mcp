## agentic-reach 가 남긴 후보 (2026-07-23)

우선순위는 사용자 소유. 에이전트는 후보와 근거만 적는다.

1. **홀드아웃 주제를 에이전트가 고르지 않는 절차** — agentic-reach 홀드아웃이 `pass^5` 100% 로
   변별력을 잃었다. 원인은 6/10 을 에이전트가 주제까지 골랐고 고를 때 도구로 존재를 확인한 것.
   유출 탐지기는 문자열만 보므로 **구성에 의한 오염**을 못 잡는다. 자를 다시 못 믿게 되는
   가장 직접적인 위험이라 최우선.
2. **AR3(어휘 공백 경고) 유지 여부** — `SR@1` −8.3%p vs `pass@3` +5%p. 경고 정밀도 75% 라
   성공한 검색 넷 중 하나에 재질의를 시킨다. 문턱을 올리거나(오탐↓ 재현율↓) 되돌리거나 유지.
   **제품 판단이라 사용자 결정 필요.** 현재 유지 중.
3. **`as_of` 가 법령ID 를 못 받는다** — `search_law` 는 `law_id` 를 주는데 `as_of` 는 법령명을
   요구해 검색→시점조회 경로가 끊긴다. 실 MCP E2E 에서 적발.
4. **본법 vs 시행령 정답 라벨** — d05·d09 가 3회 중 일부만 맞는데 오답이 전부 시행령이다.
   "대통령령으로 정한다" 위임 지점에서 정답을 어느 쪽으로 볼지 자체가 쟁점 — 라벨 규약이 먼저.
5. **벤더 교차 측정** — 블라인드 arm 이 전부 Claude 세션이었다(Codex 서브에이전트는 샌드박스
   네트워크 차단). "우리 모델에서만 되는 도구인가"가 미측정으로 남았다.

## trap-free 가 남긴 후보 (2026-07-23)

1. **본법/시행령 진동을 도구가 못 알려 준다** — 규약을 정해도 에이전트가 회차마다 본법·시행령을
   오간다(d05·d09 실측). `delegated_to` 는 이미 응답에 실려 있는데 소비층이 결론에 반영하지
   않는다. 라벨이 아니라 전달의 문제 — 측정 가능한 A/B 가 필요하다.
2. **상류 5xx 오분류** — 잘못된 인증값에 상류가 경로별로 200+오류본문 / 5xx 로 갈리고, 후자는
   "일시 장애(재시도 가능)"가 되어 인증 문제인 사용자가 영원히 재시도한다. 5xx 시 검증용
   재조회로 재분류하는 근본 수리.
3. **에이전트 기준선 재측정** — 코퍼스가 94 topic 이 됐는데 유효한 기준선은 AR2 의 dev 20건뿐.
   도구를 고치는 다음 horizon 에서 before/after 로 함께 잰다(하네스 준비 완료:
   `evidence/bench/2026-07-23-tf2-baseline/`).
4. **미개봉 문제 확보** — 지금 0개다. 다음에 문제를 늘릴 때 ADR 0002 절차로 일부를 처음부터
   떼어 두고, 그 시점에 봉인 강제 코드를 만든다.
5. **단발 도달 MISS 목록 검토** — dev 신규 맥락 26건이 단발 검색으로 정답 법령에 못 닿는다.
   맥락이 모호한 것인지 어휘 공백이 큰 어려운 케이스인지 사람이 하나씩 판단할 대상
