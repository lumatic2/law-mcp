# HORIZON — 세법을 끝까지 (tax-vertical)

> 생성: 2026-07-21 · ROADMAP marker: `harness:goal id="tax-vertical"` · 상태: **설계(승인 대기)**
> cascade 중간(Horizon). 위계: Objective(`OBJECTIVE.md`) → **Horizon**(이 문서) → Milestone(`plans/<date>-<milestone-id>-<slug>.md`) → Step.
> 진행 상태의 정본은 `ROADMAP.md` marker — 이 문서의 체크박스는 milestone boundary 에서만 동기화.

## 목표

**넓이를 1%p 씩 미는 대신, 한 분야를 끝까지 판다. 첫 분야는 세법이다.**

직전 horizon 은 전 도메인 recall 을 76%→88%(dev)·93.3%(holdout)로 올렸다. 그 축은 이제
수확체감이다 — 행정 도메인 A/B 는 **이득 0 / 손실 0** 이었고(2026-07-21 진단), 남은 개선은
upstream 랭킹과 싸우는 일이 됐다. 반면 세법에는 **아직 연결조차 안 한 자료원**이 있다.

| | 구 축 (upstream-delivery) | **신 축 (이 horizon)** |
|---|---|---|
| 폭 | 전 도메인 얕게 | **한 분야 깊게** |
| 답의 단위 | 조문 하나 | **인용 체인** — 조문 + *그 연도의* 시행일자 + 예규 + 심판례 |
| 개선 수단 | 더 나은 랭킹 엔드포인트 | **없던 층을 채운다**(심판례·예규·시점축) |
| 실패 모드 | 못 찾는다 | **찾았는데 틀린 연도 / 구속력 없는 자료를 법처럼 준다** |

이 축에서 이기는 것이 랭킹 튜닝보다 확실한 이유: 심판례·예규·시점축은 **만들면 확실히 생기는
능력**이라 A/B 에서 질 수가 없다. 반면 랭킹 개선은 방금 이득 0 을 실측했다.

그리고 세법은 **골격이 복제된다.** "법령 + 구속력 등급 다른 해석자료 + 시점축"은 노동(노동위)·
부동산에 그대로 옮겨진다. 범용 recall 튜닝은 복제되지 않는다.

실무자가 물을 법한 세법 질의 4건 중 **2건 실패**:

| 쿼리 | 정답 | 결과 |
|---|---|---|
| 접대비 손금불산입 한도 | 법인세법 §25 | ✅ 개정 용어까지 흡수(접대비→기업업무추진비) |
| 1세대 1주택 비과세 요건 | 소득세법 §89 + 시행령 §154 | ✅ |
| 과세표준 신고 후 경정청구 기한 | 국세기본법 §45조의2 | ❌ 1위 국제조세조정법, 정답에 `ai_articles` 없음 |
| 세금계산서 지연발급 가산세 | 부가가치세법 §60②1 | ❌ **부가세법이 후보에 아예 없음** |

## 미리 쓰는 실패 회고 (프리모템)

> 이 horizon 이 끝났는데 **실패했다. 왜?**

**① 평가 세트가 우리 도구를 닮았다.** 세트를 우리가 만들고 우리가 채점하니, 지금 이미 잘 되는
질의만 담겨 시작부터 90% 가 나오고 개선할 게 없어 보였다. 세법 결함은 그대로인데 숫자만 좋다.
→ **예방**: TV1 DoD — 라벨은 도구를 먼저 두드려서 만들지 않는다. 실무 유형 분포(domain-sources §2)
에서 질의를 뽑고 정답은 **조문 직접 확인**으로 고정한다. 오늘 실패한 2건(경정청구·세금계산서
가산세)을 반드시 포함한다. 홀드아웃은 라벨링 직후 봉인.

**② 심판례·예규를 넣었더니 노이즈만 늘었다.** "가산세" 4,688건 중 상위 3건이 쿼리와 무관하면
(본문검색 가나다순 함정 재발) 소비 LLM 이 엉뚱한 결정례를 자신 있게 인용한다. 도달률은 올랐는데
정확도는 떨어진다.
→ **예방**: TV2 DoD — 정렬 근거를 응답에 명시하고, **관련도를 보증 못 하면 주지 않는다**(F20 에서
확정한 "틀린 답보다 없는 답이 낫다"). 기여도 게이트를 통과 못 하면 미채택으로 닫는다.

**③ 예규를 법조문처럼 인용했다.** 예규는 법원을 구속하지 않는 "자료"인데(대법원 1998.9.8),
우리가 조문과 같은 모양으로 출하해 소비 LLM 이 근거로 삼았다. 사용자는 틀린 근거로 신고했다.
→ **예방**: TV2 에 **구속력 등급 표시를 독립 changeset 으로** 둔다(부가 기능이 아니라 게이트).
사전답변(구속력 있음)과 서면질의(없음)를 구분해 싣는다. 닫는 기준 4 에 100% 표기를 박는다.

**④ 시점 축이 반만 붙었다.** `search_law` 는 연도를 받는데 `get_law_article` 은 현행을 주고,
게다가 `target=law&efYd=…` 는 인자를 **조용히 무시**한다(실측 확인). 사용자가 섞어 쓰자 아무
경고 없이 현행 조문이 2023년 답으로 나갔다.
→ **예방**: TV3 DoD — 시점을 받는 **모든** 표면이 응답에 시행일자를 싣는다. `law&efYd` 무시
함정을 실패 프로브로 고정한다. 닫는 기준 3 에 "조용한 현행 반환 0건"을 명시.

**⑤ 세법은 깊어졌는데 나머지가 얕아졌다.** 세법 튜닝 규칙이 일반 도메인 recall 을 깎았다.
→ **예방**: 닫는 기준 5(구 dev 셋 88% 유지)를 **모든 채택 A/B 의 손실 0 조건**으로 승격한다.
