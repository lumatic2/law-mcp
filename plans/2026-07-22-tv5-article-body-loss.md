## Objective → horizon → milestone → step (위계)

- **Objective**: 한국 사람들이 '법' 관련 작업을 AI 에이전트로 할 때 설치하게 되는 MCP 의 대표 중
  하나가 된다 (← `OBJECTIVE.md`)
- **horizon**: 세법을 끝까지 (← `plans/horizons/tax-vertical.md`)
- **milestone**: TV5 — **조문 본문이 조용히 사라지고 있다.** `항내용` 이 중첩 배열
  (`[[본문, img, 표행…]]`)인 조문에서 `pickString` 이 `null` 을 돌려 **그 항의 본문이 통째로**
  빠진다. 그래서 소득세법 제55조가 "다음의 세율을 적용하여"라고 해 놓고 **표가 없다** —
  `1,400만원`·`6퍼센트`·`84만원` 이 하나도 안 나간다. 오류는 안 나고 조문은 정상처럼 반환되므로
  소비 LLM 은 "세율이 법에 없다"고 읽는다. 실측 영향: 소득세법 조문 326개 중 배열형 11개·
  표 포함 22개, 표 문자열 노드 42곳(`항내용`·`호내용`·`목내용` 3층).
  규모 근거: 추출 수리와 회귀·판정이 독립 changeset 2, 통합검증 = 세율 질의에서 표 도달 + 회귀 0.

## 스캐폴딩 결정

**① 범용 코어 3**
- source-of-truth: 이 레포. 배포는 push → 배포 사본 build.
  ⚠ **응답 본문이 길어지므로 MCP 재시작이 필요**하다(TV2·TV3 부채와 함께 한 번에 알린다).
- 검증: `npm test` + 실 API 로 세율표 도달 관측 + 교차 A/B 회귀 + dist 스모크.
- 배포/운영: 도구 개수·인자 불변. 응답 **필드**는 안 늘고 기존 필드의 **내용**이 채워진다.

**② 자기선언 도메인**
- **원문 충실성**: 표는 upstream 이 준 ASCII 박스 그대로 싣는다. 재조판하지 않는다 —
  숫자·구간을 우리가 다시 그리면 그 자체가 새 오답 경로다(F20 계열).
- **`<img>` 처리**: 표와 함께 이미지 링크가 온다. 이미지 태그는 **본문에서 제거**하되
  링크를 잃지 않게 할지는 step-1 에서 실측 후 결정 — 기본은 **태그 제거**(기존 `stripTags` 규율).
- **두 경로를 함께 고친다**: 조문 인덱스(`src/article-index.ts`)와 단건 조회 경로
  (`lawgo-provider.ts` 의 조문 추출)가 **별도 구현**이다. 한쪽만 고치면 표면마다 답이 달라진다.
- **응답 크기**: 표가 실리면 본문이 길어진다. 조문 1건 응답이 과도하게 커지지 않는지 실측하고,
  상한이 필요하면 **자르지 말고** 사실을 밝힌다(조용한 절단 금지 — TV4 교훈).
- **회귀 방어**: 본문이 길어지면 본문 기반 채점(`scoreArticles`)의 점수 분포가 바뀐다 →
  순위가 흔들릴 수 있다. **범용·세법 양쪽 교차 A/B 로 손실 0 을 확인**한다.
- 검토 후 제외: 화면·디자인 · 데이터 스토어 · 인증 · 관측 · 신규 도구 · 새 자료원 · 크레덴셜.

**③ 제외 자기점검**
- 검토 후 제외: 위 ② 마지막 줄. 특히 **표 파싱·계산** — 세율을 계산해 주고 싶어지면 그건 TV5 가
  아니다(그리고 아마 이 MCP 의 일도 아니다).
