## 결함 — 조문 본문이 조용히 사라지고 있었다

`get_law_article("소득세법","제55조")` 이 "다음의 세율을 적용하여"라고 해 놓고 **정작 세율표가
없었다.** 오류는 안 나고 조문은 정상처럼 반환되므로, 소비 LLM 은 "세율이 법에 없다"고 읽는다.

원인이 **둘**이었다. 하나는 우리 버그, 하나는 upstream 동작이다.

### ① 배열형 내용을 버리고 있었다 (우리 버그)

법제처는 `항내용`·`호내용`·`목내용` 을 문자열로 줄 때도 있고 **중첩 배열**로 줄 때도 있다:

```json
"항내용": [[ "①거주자의 종합소득에 대한 …", "<img src=…>", "┌────┬────┐", "│종합소득│세율│" ]]
```

`pickString` 은 `typeof value === "string"` 일 때만 값을 받으므로 배열이면 `null` 을 돌려
**그 항의 본문이 통째로** 빠졌다 — 표만이 아니라 산문까지.

### ② `JO=` 조회는 upstream 이 표를 빼고 준다

| 조회 방식 | 소득세법 제55조 본문 | 세율표 |
|---|---|---|
| `JO=005500` (기존 경로) | 251자 | ✗ |
| 전문(JO 없음) | **1596자** | **○** |

①만 고쳐도 조문 단위로 조회하는 한 표는 안 온다. 그래서 **전문을 받아 조문을 클라이언트에서
찾는다**(캐시로 반복 비용 제거).

## 결과

```
제55조(세율)
①거주자의 종합소득에 대한 소득세는 … 다음의 세율을 적용하여 계산한 금액 …
┌────────┬──────────────────────────┐
│종합소득        │세   율                                             │
├────────┼──────────────────────────┤
│1,400만원 이하  │과세표준의 6퍼센트                                  │
│1,400만원 초과  │84만원 + (1,400만원을 초과하는 금액의 15퍼센트)     │
```

## 규율

- **표는 재조판하지 않는다.** upstream 이 준 ASCII 박스 그대로 싣는다 — 숫자·구간을 우리가 다시
  그리면 그 자체가 새 오답 경로다. 그래서 정리 규칙을 조각마다 나눴다:
  **표 줄은 공백 보존**(칸 정렬이 곧 의미), 산문만 접는다.
- **두 표면이 같은 함수를 쓴다.** 조문 인덱스(`article-index.ts`)와 단건 조회
  (`lawgo-provider.ts`)가 별도 구현이라, 한쪽만 고치면 표면마다 답이 갈린다 — 테스트로 고정.
- 이미지 태그는 본문에서 제거(기존 `stripTags` 규율 유지).
