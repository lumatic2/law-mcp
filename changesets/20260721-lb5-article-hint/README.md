# LB5 step-3 — 조문 축: 연계 조문 출하 + 재정렬 기각

- 일자: 2026-07-21 · milestone: LB5 · 증거: `evidence/bench/2026-07-21-lb5-article-sweep.md`

## 이 step 이 드러낸 것 — assisted 지표는 **출하되지 않는 경로**를 재고 있었다

`scoreArticles` 는 `bench/` 와 테스트에서만 import 된다. **어떤 MCP 도구도 호출하지 않는다.**
따라서 "assisted acc@3 62.5%" 는 소비자가 실제로 받는 값이 아니라 벤치 내부 계산이었다.
이 사실이 이 step 의 방향을 바꿨다 — 점수 함수를 튜닝하면 **벤치 숫자만** 좋아진다.

## 채택: 연계 조문을 `search_law` 응답에 실어 보낸다

`SearchLawItem.linked_articles` 신설. 부스트가 발동한 법령에 그 용어가 실제로 쓰인 조문 번호를
최대 5개까지 함께 반환한다. **추가 호출 0** — step-2 에서 이미 손에 들어온 데이터다.

```
"부당해고 구제신청 기간" → 근로기준법 (linked_articles: ["제23조","제28조"])
```

소비 LLM 은 이제 법령과 조문을 한 응답에서 받는다. 지금까지 이 정보는 아예 없었다.

## 기각: 조문 점수 재정렬 (가설 5호)

연계 조문으로 조문 예측 순서를 바꾸는 7가지 배치를 dev 24건(정답 법령 주어짐)에서 비교했다.

| 경로 | acc@1 | acc@3 |
|---|---|---|
| (A) 기존 `scoreArticles` (미출하) | **50.0%** | 62.5% |
| (B) 연계 조문 단독 (출하됨) | 16.7% | 33.3% |
| (C) 연계 우선 + 기존 보완 | 29.2% | 58.3% |
| (D) 기존 우선 + 연계 보완 | 50.0% | 62.5% |
| (E) 기존 무예측일 때만 연계 | 50.0% | 62.5% |
| (F) 교차 배치 A1,B1,A2,B2 | 50.0% | 66.7% |
| (G) 연계에 2번 자리 1칸 | 50.0% | 66.7% |

**기각 사유 2가지:**

1. **이득이 노이즈와 구분되지 않는다.** 최선(F·G)이 +4.2%p 인데 이는 24건 중 **1건**이고,
   step-2 에서 같은 코드로 두 번 돌린 공식 러너가 72.0%/76.0% 로 갈렸다(=1건 노이즈).
2. **바꿀 대상이 출하되지 않는다.** 재정렬은 `scoreArticles` 를 고치는 일이고, 그 함수는
   제품 경로에 없다. 벤치 점수만 올리는 변경이다.

(C) 는 **명확히 해롭다**(acc@1 50.0% → 29.2%). 연계를 무조건 앞세우면 안 된다는 반증이 나왔다.

## 그래도 연계가 이긴 사례는 실재한다 (기각의 근거를 약화시키지 않는 선에서 기록)

`해고예고 수당 지급 의무` — 기존 경로는 제43·46·37조(전부 오답), 연계는 **제26조(정답)** 단독 적중.
집계에서 이득이 사라진 이유는 기존 예측 3개가 상위 자리를 다 채워 정답이 4번째로 밀렸기 때문이다.
`linked_articles` 로 **이 정보 자체는 소비자에게 전달**되므로, 순서를 조작하지 않고도 값은 전해진다.
