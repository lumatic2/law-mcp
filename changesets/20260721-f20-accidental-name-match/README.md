## 무엇이 달라지나

```
get_law_article("민법 시행령", "제1조")
  전: 난민법 시행령 제1조 본문 + "느슨하게 해석함" 경고
  후: LAW_NOT_FOUND — "난민법 시행령" 은(는) 이름이 우연히 겹칠 뿐 다른 법입니다.
```

UD4 에서 이 결함을 **경고로만** 덮었다. 그걸 되돌린다 — 소비 LLM 은 경고를 읽어도 조문 본문이
있으면 그대로 인용한다. **틀린 답보다 없는 답이 낫다.**

## 사고와 정당한 느슨함을 가르는 선

폴백 자체는 없앨 수 없다. 없애면 `공정거래법` → `독점규제 및 공정거래에 관한 법률` 같은
정당한 약칭 조회가 같이 죽는다. 그래서 **실측으로** 선을 그었다 (이름 해석 프로브 10건):

| 요청 | 반환 | 요청이 반환의 부분문자열인가 | 판정 |
|---|---|---|---|
| `민법 시행령` | 난민법 시행령 | **예** (앞에서 시작 안 함) | 사고 → 거절 |
| `상법 시행령` | 기상법 시행령 | **예** | 사고 → 거절 |
| `공정거래법` | 독점규제 및 공정거래에 관한 법률 | 아니오 | 정당 → 보존 |
| `형법 시행령` | 형의 집행 및 …에 관한 법률 시행령 | 아니오 | 정당 → 보존 |

**사고는 요청 문자열이 반환 이름 안에 통째로 들어 있되 앞에서 시작하지 않을 때 정확히 발생한다.**
그건 "요청한 이름의 법이 따로 없다"는 뜻이다 — `난민법` 이 있을 뿐 `민법 시행령` 은 없다.
반대로 약칭·별칭은 요청을 부분문자열로 품지 않는다(글자가 재배열되거나 늘어난다).

`부가가치세` → `부가가치세법` 은 upstream 이 `prefix` 로 표시해 애초에 이 분기에 오지 않는다.
UD4 주석은 이 폴백의 존재 이유를 `부가가치세` 로 적었는데 **틀린 근거였다** — 실측으로 정정한다.
그래도 그 표시를 못 믿는 경우를 위해 `startsWith` 후보를 먼저 뒤진다.
