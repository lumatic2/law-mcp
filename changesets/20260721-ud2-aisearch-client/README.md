# UD2 step-1 — `aiSearch` 클라이언트 + 응답 계약 고정

- 일자: 2026-07-21 · milestone: UD2 · plan: `plans/2026-07-21-ud2-aisearch-adoption.md`

## 무엇을 붙이나

법제처 `aiSearch` — **자연어 질의를 조문 단위로 답하는 검색**. 이 레포가 다섯 milestone 동안
"upstream 에 관련도 랭킹이 없다"를 전제로 후보 생성기를 직접 만들어 온 그 전제를 뒤집는 엔드포인트다.

이 step 은 **결과를 판단하지 않고 정규화만** 한다. 어떻게 섞을지는 step-2 가 실측으로 정한다.

## 응답 구조 (2026-07-21 실측 — 테스트로 핀 고정)

```
aiSearch.법령조문[] → { 법령명, 법령ID, 조문번호("0028"), 조문가지번호("00"),
                        조문제목, 조문내용, id(순위) }
```

`법령ID` 를 그대로 주기 때문에 **추가 호출 없이** `get_law_article` 경로로 이어진다.

## 방어 원칙 두 가지

1. **키 핀 고정.** upstream 키는 규칙이 없고, 바뀌어도 HTTP 200 으로 온다(LB3 `ordin` 사고).
   컨테이너·행·필드 키를 실측값으로 박고 테스트로 잠갔다.
2. **예외 누출 금지.** `aiSearch` 는 법제처 블랙박스다. 무응답·타임아웃·스키마 변경 어느 경우든
   **빈 결과**를 돌려, 호출자가 기존 사다리로 graceful degrade 하게 한다.

설계 판단 하나: **순위 필드가 없으면 배열 순서로 떨어진다.** 순위를 잃는 것이 이 채널의 존재
이유를 잃는 것이라 조용히 0 으로 두지 않는다.

### 실 API 스모크가 드러낸 것

```
공소시효 기간      → 1. 형사소송법 제249조(공소시효의 기간)
행정심판 청구 기간  → 1. 행정심판법 제27조(심판청구의 기간)
정당방위 성립 요건  → 1. 정당법 제4조  2. 방위사업법 제7조  3. 형법 제21조(정당방위)
```

**`행정심판 청구 기간` 은 UD1 기준선에서 우리 파이프라인이 검찰청법을 반환하던 쿼리다.**
`aiSearch` 는 1위로 정답을 준다.

동시에 **알려진 약점도 재현됐다** — `정당방위 성립 요건` 에 정당법·방위사업법이 1·2위다.
이것이 "대체가 아니라 병합" 결정의 근거이며, 다만 형법 제21조가 3위라 recall@3 은 살아남는다.
이 유형을 어떻게 다룰지는 step-2 의 실측 대상이다.

## Contract

- source of truth: `src/ai-search.ts`
- deploy target: 아직 없음 — **어떤 도구도 이 모듈을 호출하지 않는다.** 배포는 step-2 이후.
- compatibility: 순수 추가. 기존 경로 무변경.
- 비용 예산: 검색 1회당 추가 호출 ≤1, LRU 캐시 필수(miss 도 캐시).
- out of scope: 후보 병합(step-2) · 조문 출하(step-3)
