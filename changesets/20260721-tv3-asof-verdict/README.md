## 러너가 재는 것

시점 정확도만 재면 절반이다. **"못 맞췄는데 현행을 슬쩍 주는 일이 없는가"** 를 함께 잰다 —
그게 이 milestone 의 존재 이유이고 horizon 닫는 기준 3 이다. 두 방향으로 확인한다:

- 반환 시행일자가 **현행과 같으면서 기준일보다 뒤인** 경우를 센다(정상 경로의 조용한 현행 반환).
- 거절되어야 하는 입력 5종이 실제로 거절되는지 센다(거절 경로).

둘 중 하나라도 새면 러너가 `exit 1` 을 낸다.

## 실측이 보여준 것

반환 시행일자가 **연초가 아닌 경우가 많다**(20211230·20201210·20230401). 두 단계 조회가
제대로 돈다는 증거다 — 기준일(12/31)을 `efYd` 에 그대로 넣는 구현이었다면 전부 0 조문이었다.

  이번엔 둘 다 0 이라 발화하지 않았다.
