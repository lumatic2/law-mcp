## 무엇이 달라지나

세법 해석의 1차 자료원이 들어왔다. **법제처 `expc` 로 "가산세"를 검색하면 0건**이다 —
세법 해석 권한은 기재부·국세청에 있고 법제처 해석 대상이 아니다. 국세청 예규를 안 쓰면
세법 해석 자료원이 통째로 없다.

```
search_legal_source(source="ntsExpc", query="가산세")   → 1,938건
  기준-2023-법규법인-0191[법규과-992] | 국세청 | 2025.05.15
  원문: https://taxlaw.nts.go.kr/qt/USEQTA002P.do?ntstDcmId=200000000000012562
```

실무자가 인용하는 **안건번호 체계가 그대로** 온다. `source` enum 17 → 19(`ntsExpc`·`moefExpc`).
도구는 여전히 11개.

## 전문 조회는 없다 — 빈 응답 대신 명시 거절

plan 이 미지로 남긴 전문 계약을 실측으로 닫았다. **2경로 전부 실패**:
`target=ntsCgmExpc` 는 HTML 을 반환하고, `target=expc` 로 같은 ID 를 부르면
"일치하는 법령해석례가 없습니다"가 온다(ID 공간이 다르다).

전문을 부르면 오는 HTML 을 파싱해 빈 결과로 흘리면 소비 LLM 은 **"찾아봤지만 내용이 없다"**로
읽는다. 사실은 "여기서는 못 준다"이고 원문 링크라는 대체 경로가 있다. 그래서
`detailUnavailable` 을 두고 **upstream 을 부르지도 않고** 사유와 대체 경로를 말한다.

```
get_legal_source(source="ntsExpc", source_id="1643869")
  → DETAIL_NOT_AVAILABLE — "…전문 조회가 없다(2026-07-21 실측 — HTML 반환).
     검색 결과의 `원문링크`(국세법령정보시스템)로 원문을 확인해야 한다."
```
