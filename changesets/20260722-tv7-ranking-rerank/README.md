## 무엇을 한 신호인가

TV4 가 판정한 결함: **도달은 고쳤는데 순위가 안 움직였다.** 순위 신호가 법령명 문자열뿐이라
이름과 글자를 안 나누는 정답(`수정신고`→`국세기본법`)이 후보 풀에 들어와도 밀렸다.

`aiSearch` 는 그 값을 **이미 무료로 준다** — 검색과 **병렬로 이미 호출 중**이고 응답 행마다
`조문제목`·`조문내용`·순위가 들어 있다. 우리는 그걸 "상위 N개 법 승격"에만 쓰고 버리고 있었다.
**추가 HTTP 호출 0.**

### 대체가 아니라 신호다

`aiSearch` 자체 순서는 세법 dev 에서 **76.7%** 로 현행 83.3% 보다 나쁘다. 그 순서를 그대로
믿으면 맞히던 것을 깬다. 그래서 최종 순서는 우리가 정하고 aiSearch 는 **채점 재료**로만 쓴다.

### 새 채점기를 발명하지 않았다

LB2 의 `scoreArticles`(조문제목 ×3 + 본문, 한 조문에 토큰이 몰리면 제곱 가중)를 **그대로**
재사용한다. 여기서 새 가중치를 만들면 그건 평가 세트에 맞춘 것이지 신호가 좋아진 게 아니다.
동점이면 aiSearch 순위 → 원래 순서로 떨어진다.

## 하마터면 놓칠 뻔한 것 — 드리프트를 회귀로 오독

신호를 켜고 범용 세트를 **한 번** 돌렸더니 `recall@1` 이 80.0% → 68.0% 로 보였다. 회귀처럼
읽혔지만, 같은 세트를 **교차 측정**하니 두 배치가 **모든 쿼리에서 동일**했다(이득 0·손실 0,
@1 도 0·0). 교차 측정한 control 자체가 @3 84.0% 로 나온다 — 즉 단일 측정 수치가 떠다닌 것이다.

**이 벤치는 노이즈가 아니라 드리프트가 문제**라는 F13 규율이 오판을 막았다. 단일 측정으로
채택·기각을 판정하지 않는다.

## 규율

- **과적합 방어(F5)**: 모듈에 법명·도메인·쿼리 토큰이 **하나도 없다**(테스트가 한글 리터럴 금지).
- **신호가 없으면 순서를 흔들지 않는다**: aiSearch 가 비었거나·실패했거나·어떤 후보도 점수를
  못 얻으면 원본 그대로. `lookupAiSearch` 가 모든 실패를 빈 결과로 흡수한다.
- **후보 풀 밖의 법을 새로 끼워 넣지 않는다** — 그건 병합(`mergeAiSearch`)의 일이다.
- 기본값은 **판정을 통과한 뒤에** 켰다(UD2 와 같은 규율 — 판정 전엔 false).
