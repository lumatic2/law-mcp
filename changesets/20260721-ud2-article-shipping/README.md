# UD2 step-3 — 조문을 제품 응답으로 출하 (F4 해소)

- 일자: 2026-07-21 · milestone: UD2 · plan: `plans/2026-07-21-ud2-aisearch-adoption.md`
- 증거: `evidence/bench/2026-07-21-ud2-final.json`

## 무엇이 달라지나

`search_law` 가 "어느 법인지"에서 멈추지 않고 **"몇 조인지"까지 준다.** upstream 이 이미
조문 단위로 답했으므로 **추가 호출은 0**이다.

```
search_law("행정심판 청구 기간")
  → 행정심판법  ai_articles: [제27조 "심판청구의 기간", …]
```

## 출처를 섞지 않는다

`ai_articles`(관련도 — 이 질문에 답하는 조문)와 `linked_articles`(용어 색인 — 그 용어가 쓰인
조문)는 **다른 신호**다. 한 필드에 합치면 소비 LLM 이 무엇을 받았는지 모른다. 해당 없으면
필드 자체를 달지 않는다(빈 배열 오염 금지).

## ★ 측정 대상을 바꿨다 — F4 의 실체

LB2~LB5 는 조문 축을 `scoreArticles` 로 쟀다. 그런데 그 함수의 결과는 **제품이 주는 값이
아니었다** — 벤치 전용 경로였고, 사용자는 그 조문을 받은 적이 없다. 이번에 러너를
**응답에 실제로 실린 값**만 보도록 바꿨다.

그래서 이제 조문 축에 두 수치가 나온다:
- **조문 출하율 76.0%** — 조문 라벨 25건 중 19건에 조문이 실렸다
- **조문 정확도 73.7%** (n=19) — 실린 것 중 1위가 정답인 비율

정확도만 보면 "안 실린 6건"이 분모에서 빠져 좋아 보인다. 두 수치를 같이 찍는 이유다.
