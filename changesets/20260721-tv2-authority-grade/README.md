# TV2 step-3 — 구속력 등급 (예규를 법조문처럼 인용하지 못하게)

- 일자: 2026-07-21 · horizon: `tax-vertical` · milestone: TV2
- plan: `plans/2026-07-21-tv2-tribunal-and-rulings.md` step-3
- 근거: `research/2026-07-21-tax-domain-tax-practice-sources.md` §1

## 왜 부가 기능이 아니라 게이트인가

도메인 리서치가 확인한 사실: **예규는 법원(法院)을 구속하지 않는 "자료"** 이고(대법원 1998.9.8
취지), 예규문 자체에 "사실관계가 다르면 답변이 달라질 수 있다"는 면책 단서가 붙는다.

그런데 우리가 조문과 같은 모양으로 출하하면 소비 LLM 은 그걸 근거로 인용한다. **F20 과 같은
부류의 함정**이다 — 그때는 이름이 우연히 겹친 법을 자신 있게 줬고, 여기서는 구속력 없는 자료를
법처럼 준다. F20 에서 확정한 원칙이 그대로 적용된다: **틀린 답보다 없는 답이 낫다.**

## 무엇이 달라지나

모든 법원(法源) 응답에 `authority` · `authority_note` · `data_as_of` 가 실린다.

```
search_legal_source(source="ntsExpc", query="가산세")
  authority: "reference_only"   data_as_of: "2025.06.28"
  note: "국세청 법령해석(예규)은(는) 법령이 아니라 과세관청의 해석이다. 법원을 구속하지 않고,
         예규문 자체에 \"사실관계가 다르면 답변이 달라질 수 있다\"는 단서가 붙는다.
         근거로 인용하지 말고 법령·판례·심판례로 되짚어야 한다."
  warn: ⚠ 국세청 법령해석(예규): …  ← 참고자료 등급만 경고로 한 번 더

search_legal_source(source="ttDecc", query="이행강제금")
  authority: "adjudication"     data_as_of: "2024.12.18"
  (경고 없음 — 과잉 경고는 경고를 죽인다)
```

등급 5종: `adjudication`(심판례·위원회 판정) · `reference_only`(예규·해석례) ·
`constitutional`(헌재) · `statute`(자치법규) · `dictionary`(용어).

**전부 `adjudication` 으로 뭉개지 않았다.** 헌재결정은 법률의 효력을 없애고, 자치법규는 그
자체가 법규이며, 용어는 법적 판단의 근거가 아니다 — 서로 다른 무게를 다른 등급으로 둔다.

## 강제 방식 — 타입으로 막았다

`SourceDescriptor.authority` 와 `SearchLegalSourceResult.authority` 를 **선택이 아니라 필수**로
뒀다. 등급 없는 자료원을 추가하면 컴파일이 깨진다(실제로 이 변경에서 기존 8개 descriptor 가
전부 컴파일 오류로 걸렸고, 그게 의도한 강제다).

## 데이터 기준일도 함께 싣는다

심판례 2024.12.18 · 예규 2025.06.28. **"최근 결정례"에는 구조적으로 못 답한다.** 숨기면 소비
LLM 이 오래된 목록을 최신으로 읽는다. upstream 이 준 값을 그대로 싣고 추정하지 않는다
(값이 없는 자료원은 `null`).

## 검증

- `npm test` **218/218** (신규 7건 — 전 자료원 등급 존재, 예규가 "법령이 아니라"·"구속하지 않"을
  명시, 법제처 해석례도 참고자료, 심판례/위원회는 재결, 헌재/자치법규/용어 개별 등급,
  **강한 등급이 함부로 늘지 않음**(보수 판정 규율), 예규는 전문없음+참고자료를 함께 가짐)
- `npx tsc --noEmit` 통과
- 실 API 관측: 위 3자료원의 등급·기준일·경고 발화 — 예규만 경고가 붙고 심판례는 안 붙는다
- **과잉 수리 방지**: 전문이 있는 자료원이 막히지 않는 것을 step-2 테스트가 계속 고정

## 남는 것

- 세법해석 **사전답변**은 과세관청을 구속하지만(신뢰보호), 검색 목록에서 서면질의와 구분되지
  않는다. 그래서 **보수적으로 전부 `reference_only`** 로 둔다 — "모르면 구속력 있다"로 새면
  그게 오답의 원천이다. note 에 이 사실을 명시했다.
