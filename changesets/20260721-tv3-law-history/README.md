# TV3 step-3 — 연혁 목록 (`include_history`)

- 일자: 2026-07-21 · horizon: `tax-vertical` · milestone: TV3
- plan: `plans/2026-07-21-tv3-tax-year-axis.md` step-3

## 무엇이 달라지나

"이 조문이 언제 바뀌었나"에 답한다. `search_law(query, include_history=true)` 를 주면
**1위 법령**의 시행 연혁이 함께 온다.

```
search_law("소득세법", include_history=true)
  1위: 소득세법 · 연혁 100건
  구분: 시행예정 7 · 현행 1 · 연혁 92
  최근 3: 20280101(시행예정) 20270101(시행예정) 20270101(시행예정)
  경고: '소득세법' 의 시행 연혁 100건을 함께 실었다 — 시행예정/현행/연혁이 섞여 있으니
        `현행연혁코드` 로 구분할 것.
```

**경고가 필요한 이유**: 목록 첫 항목이 `시행예정` 이다. `현행연혁코드` 를 안 보고 첫 항목을
쓰면 **아직 시행되지 않은 법을 현행으로 오해**한다.

## 기본 꺼짐 — 안 묻는 검색에 비용을 얹지 않는다

연혁은 API 호출을 1 늘린다. 시점을 묻지 않는 대다수 검색이 그 비용을 낼 이유가 없으므로
`include_history` 는 **기본 false** 다(plan 비용 예산: 시점 미지정 경로 호출 증가 0).
1위 항목에만 붙인다 — 전 항목에 붙이면 호출이 결과 수만큼 늘어난다.

## 연혁 경로는 `eflaw` 하나뿐이다

`lsHistory` 는 HTML, `lsHstInf` 는 0건, `eflawJo` 는 빈 응답이다(2026-07-21 실측).
다른 target 으로 바꾸면 **조용히 빈 연혁**이 된다. 테스트가 죽은 target 3종의 사용을 거부한다.
