## 라벨링 방향 — 도구를 먼저 두드리지 않았다

프리모템 ①(세트가 우리 도구를 닮는다)의 예방장치다. 질의는 **실무 유형 분포에서 먼저** 뽑고,
정답은 **조문을 직접 열어** 고정했다. 도구가 지금 못 찾는 질의가 세트에 남는 것이 정상이며,
그게 이 세트의 존재 이유다. 실패 2건은 테스트로 **잠갔다**:

- `과세표준 신고 후 경정청구 기한` → 국세기본법 제45조의2 (실측 1위는 국제조세조정법)
- `세금계산서 지연발급 가산세` → 부가가치세법 제60조 (실측에서 부가세법이 후보에 아예 없었다)

성공 2건(`접대비 손금불산입 한도`·`1세대 1주택 비과세 요건`)도 회귀 고정용으로 담았다.

## 이 세트가 첫날에 잡은 것 (finding — TV1 에서는 고치지 않는다)

`verify-labels` 의 ① 법령명 실재 확인이 **하위법령에서 흔들린다.** 라벨 오류가 아니라 도구 결함이라
plan 범위상 수리하지 않고 남긴다(TV1 은 `src/` 무변경).

**F21 — 본법 승격이 `match_type: "exact"` 를 참칭한다.** 실측:

```
searchLaw("소득세법 시행령")
  exact    001565 소득세법          ← 1위. 요청한 법이 아닌데 exact 다
  exact    003956 소득세법 시행령    ← 진짜 정답이 2위
```

UD4 의 본법 승격은 경고로 사실을 밝히지만(`'소득세법 시행규칙' 자리에 근거 본법을 놓음`),
**`match_type` 까지 `exact` 로 표기**한다. `match_type === "exact"` 를 신뢰하는 소비자는
시행령을 물었는데 본법을 집는다 — **F20 과 같은 부류(자신 있게 틀린 라벨)** 다.

**F22 — 하위법령이 용어 연계 노이즈에 밀린다.** 실측:

```
searchLaw("법인세법 시행령")  →  법인세법 / 법인세법 시행규칙 / 전기사업회계규칙 /
                                조세특례제한법 / **법인세법 시행령(5위)**
```

`전기사업회계규칙`·`조세특례제한법` 은 '법인세법' 용어 연계로 올라온 것이다. 정확한 법령명을
통째로 줬는데 그 법이 5위다. TV4(도달 결함)의 입력으로 넘긴다.

> 두 결함 모두 **조문 조회 경로(`get_law_article`)에서는 재현되지 않는다** — 위 50건이 전부
> 정상 조회됐다. 검색 경로 한정이다.
