# UD4 step-2 — 보조 채널을 사다리와 동시에 (F19)

- 일자: 2026-07-21 · milestone: UD4 · plan: `plans/2026-07-21-ud4-parent-law-and-ladder.md`

## 진단이 계획과 달랐다

계획은 "`aiSearch` 가 답했으면 사다리 하위 단계를 생략"이었다. 코드를 열어 보니 **사다리는
이미 첫 단에서 결과가 나오면 멈춘다** — 생략할 게 없었다.

실제 꼬리의 원인은 **용어 연계가 사다리 뒤에 순차로 붙어 있던 것**이었다. UD2 에서 `aiSearch`
에 대해 고친 것과 같은 구조 결함인데, 용어 연계는 그때 손대지 않았다. 연계도 사다리 결과에
의존하지 않으므로 동시에 띄운다.

**품질 중립이 설계로 보장된다** — 결과를 바꾸지 않고 캐시만 미리 채운다. 부스트 본체는 그
캐시에서 같은 값을 읽는다. 토큰 선정 로직은 프리페치와 본체가 **같은 함수**를 쓴다(복제하면
한쪽만 바뀌었을 때 프리페치가 조용히 헛돈다).

## 지연

검색 1회 중앙 **2939ms → 1966ms**.

최대값은 **판정하지 않는다.** `지상권 존속기간` 이 3956ms~9799ms 로 널뛰는데, 승격을 **끈**
상태가 오히려 9799ms 였다. 우리 구조가 아니라 upstream 변동이 지배하는 구간이라, 이 수치로
개선/악화를 말할 수 없다. DoD 의 "최대 감소"는 **판정 불가**로 기록한다.
